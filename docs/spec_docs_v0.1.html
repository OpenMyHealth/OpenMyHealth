2.2
Pipeline

암호화 저장 SYSTEM

파싱 → FHIR R4 정규화 → AES-256-GCM 암호화 → IndexedDB. 원본 파일도 암호화 보관.

단계동작
1. 텍스트 추출PDF.js (PDF) / Tesseract.js WASM (이미지). 메모리에서만 처리.
2. 문서 분류WASM BERT 모델 (~30MB). Multi-label. → Section 2.2 상세.
3. 타입별 파싱분류 결과에 따라 전용 파서 실행 (lab/rx/dx/report/note).
4. FHIR 정규화Observation, MedicationStatement, Condition, DiagnosticReport, DocumentReference.
5. 임베딩MiniLM WASM (384차원). 의미 검색용. 임베딩 진행률 상세는 v1.1 (F-18).
6. 암호화AES-256-GCM. PIN → PBKDF2 → derived key. IV per-record.
7. 저장IndexedDB. schema_version 필드 포함 (F-22 마이그레이션용).
Security
모든 처리가 브라우저 내부. 서버 전송 0건. Tesseract WASM도 로컬 실행. 암호화 키는 메모리에만, 탭 닫기 시 폐기.
Section 2.2 — New in v1.4

Document Classification Pipeline

WASM 기반 문서 분류. 서버 전송 없이 브라우저에서 문서 타입 결정.

텍스트 추출 (PDF.js / Tesseract.js) v WASM BERT 분류 모델 (~30MB, 최초 1회 캐시) | Multi-label Classification | 예: "혈액검사 + 처방전" 동시 태그 가능 v 분류 결과 → 타입별 전용 파서 | lab_parser → Observation (검사 수치) | rx_parser → MedicationStatement (처방약) | dx_parser → Condition (진단명) | report_parser → DiagnosticReport (보고서) | note_parser → DocumentReference (진료기록) v FHIR R4 정규화 → 임베딩 → 암호화 → IndexedDB
구성요소상세
모델Lightweight BERT (DistilBERT 변형), ~30MB WASM 바이너리. ONNX Runtime Web.
분류 방식Multi-label. 하나의 문서에 여러 타입 태그 가능 (예: 종합검진 = lab + dx + report).
캐싱최초 로드 시 Cache Storage에 저장. 이후 오프라인에서도 작동.
FallbackWASM 미지원/실패 시 → 규칙 기반(regex) 분류기로 폴백. 정확도 낮지만 동작 보장.
성능 기준v1.1에서 정의 (F-23). 목표: 1페이지당 ≤ 3초 (분류+파싱 합산).
프라이버시모든 추론이 브라우저 내 WASM. 모델 바이너리만 CDN에서 다운로드 (1회).
Implementation
ONNX Runtime Web (WASM backend). Web Worker에서 실행하여 UI 블로킹 방지. SharedArrayBuffer 사용 가능 시 멀티스레드.

Section 4

OAuth & Session Binding

MCP 연결의 인증 흐름. Stateless JWT, 서버 저장 없음.

4.0
Auto

OAuth 2.0 Authorization Code Flow

ChatGPT가 Extension의 OAuth endpoint에 요청 → 토큰 발급 → MCP 세션 수립.

ChatGPT → MCP Server (Extension Relay) | GET /authorize?client_id=...&redirect_uri=...&state=... v Extension Relay → ChatGPT callback | code=AUTH_CODE v ChatGPT → POST /token | code=AUTH_CODE&grant_type=authorization_code v Extension Relay → JWT 발급 | access_token (stateless JWT, exp: 1h) | refresh_token (exp: 30d) v ChatGPT ↔ MCP Server (Streamable HTTP) | Authorization: Bearer {JWT} | tools/list → resources/read
항목Spec
JWTStateless. 서버에 토큰 DB 없음. Payload: {sub, iat, exp, scope}.
Access Token만료: 1시간. 만료 시 refresh_token으로 갱신.
Refresh Token만료: 30일. 1회용 rotation.
Relay ServerOAuth + MCP 메시지 중계만. 데이터 저장/로깅 없음. 패스스루.
